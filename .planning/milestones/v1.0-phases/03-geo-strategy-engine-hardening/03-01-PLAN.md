<objective>
Fix null value handling across the GEO strategy engine so that missing metadata values (None, "NA", "null", etc.) are consistently treated as falsy, preventing incorrect pipeline selection.

Purpose: The root cause of wrong pipeline selection is that `_sanitize_null_values` converts None to the truthy string "NA", which cascades through `has_file()`, `data_availability`, rule evaluation, and `_derive_analysis()` -- all using `bool()` truthiness that treats "NA" as True. This plan fixes the producer (sanitizer) and all consumers (null checks) with a shared helper.

Output: All null-like values are consistently falsy throughout the strategy engine. TDD: tests written first describing correct null behavior, then implementation to make them pass.
</objective>
