Tests to write:

1. **TestV2ViewType** -- Create a minimal tea.Model whose View() returns tea.View via tea.NewView("hello"). Verify the model satisfies the tea.Model interface (compile check). Call View() and check the returned type is tea.View.

2. **TestV2ColorType** -- Call lipgloss.Color("#e45c47") and verify it returns a value assignable to color.Color (image/color standard library). Create a lipgloss.NewStyle().Foreground(lipgloss.Color("#e45c47")) and call Render("test") -- must not panic.

3. **TestV2KeyPressMsg** -- Construct tea.KeyPressMsg values for: enter (Code: tea.KeyEnter), ctrl+c (Code: 'c', Mod: tea.ModCtrl), regular char 'a' (Code: 'a', Text: "a"), escape (Code: tea.KeyEscape), space (Code: tea.KeySpace). For each, verify msg.String() returns the expected string ("enter", "ctrl+c", "a", "esc"/"escape", "space").

4. **TestV2ViewportBasic** -- Create a viewport.Model, set Width/Height, set Content to a multi-line string, call View(). Verify output is non-empty and does not panic.

5. **TestV2GlamourCoexistence** -- Import glamour (v1 path: github.com/charmbracelet/glamour) AND lipgloss v2 (charm.land/lipgloss/v2) in the same test. Render markdown with glamour. Create a lipgloss v2 style. Verify both work without conflict. This proves the module coexistence claim.

6. **TestV2ProgramOptions** -- Verify tea.NewView has AltScreen and MouseMode fields (compile check by setting them). This confirms WithAltScreen/WithMouseCellMotion move to View fields.

After creating the test file, run `go mod tidy` from lobster-tui/ to pull in any needed v2 dependencies.
