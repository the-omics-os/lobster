<objective>
Add partial parse signaling to GEO chunk parser and propagate through all call sites.

Purpose: The chunk parser silently truncates results when hitting memory limits (breaks at line 724, concatenates at line 753 with no signal). Callers have no way to distinguish complete data from truncated data. This plan introduces a ParseResult dataclass that carries integrity metadata (is_partial, rows_read, truncation_reason) and updates all consumers.

Output: ParseResult dataclass in parser.py, updated parse_expression_file and parse_supplementary_file return types, all 5 geo_service.py call sites handling partial results with logged warnings.
</objective>
