**Assignment pattern (for tools inside factory closures):**
Since tools are defined inside the factory function as closures, metadata must be assigned AFTER the `@tool`-decorated function definition, before the next tool definition. Example:

```python
    @tool
    def cluster_cells(modality_name: str, ...) -> str:
        """..."""
        # implementation
        ...

    # AQUADIF metadata
    cluster_cells.metadata = {"categories": ["ANALYZE"], "provenance": True}
    cluster_cells.tags = ["ANALYZE"]

    @tool
    def subcluster_cells(...) -> str:
        ...
```

For shared_tools.py, tools are also closures inside `create_shared_tools()`, so the same pattern applies.

**CRITICAL ordering rules:**
1. Metadata assignment MUST be AFTER the function definition (after `@tool` decorator has processed it)
2. Both `.metadata` and `.tags` MUST be set to the same category list
3. Primary category (first in list) determines provenance requirement
4. Do NOT add `from lobster.config.aquadif import AquadifCategory` at module level — keep it inside the factory. Actually, since we're using string literals ("ANALYZE", "IMPORT", etc.) in the metadata dicts (not enum values), the AquadifCategory import is not strictly needed in these files. String literals are acceptable per the research doc and match what contract tests validate.
