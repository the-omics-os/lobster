Create `skills/lobster-dev/references/aquadif-migration.md` with dual format per user decision: migration checklist (process) + annotated code examples (pattern).

**Document structure:**

1. **Header** — Title, purpose (migration guide for adding AQUADIF to existing agent packages), audience (coding agents in Phase 4 rollout + human contributors)

2. **Migration Checklist** (step-by-step process):
   - Step 1: Inventory tools — count @tool-decorated functions in agent file(s) and shared_tools.py
   - Step 2: Categorize each tool — use the category decision flowchart from aquadif-contract.md. Apply "single category preferred" philosophy and 80% rule. Primary = purpose, not pipeline order.
   - Step 3: Determine provenance flags — check primary category against PROVENANCE_REQUIRED set (7 of 10 require provenance: IMPORT, QUALITY, FILTER, PREPROCESS, ANALYZE, ANNOTATE, SYNTHESIZE)
   - Step 4: Add metadata inline — use 2-line assignment pattern after each @tool function. Both .metadata and .tags must be set identically.
   - Step 5: Create contract test file — inherit from AgentContractTestMixin, one class per agent in the package, set is_parent_agent appropriately
   - Step 6: Run contract tests — `pytest -m contract <test_file> -v`
   - Step 7: Run existing tests — confirm no backward compatibility regressions

3. **Code Patterns** (annotated examples):
   - Single-category tool with provenance (IMPORT example)
   - Single-category tool without provenance (UTILITY example)
   - Multi-category tool (PREPROCESS + FILTER example)
   - Contract test class setup (mixin inheritance)
   - Include the "what NOT to do" anti-patterns: metadata before @tool, only setting one of .metadata/.tags, wrong primary order

4. **Transcriptomics Worked Example** (the actual mapping table from Plan 01):
   - Full table of all 22 tools with their assigned categories, provenance flags, and rationale for non-obvious mappings
   - Read the actual metadata from Plan 01 summary or the source files themselves to get the real mapping
   - Include rationale for the 4 ambiguous tools identified in research: filter_and_normalize, merge_sample_metadata, convert_gene_identifiers, prepare_bulk_for_de
   - Omit rationale for obvious mappings (e.g., cluster_cells = ANALYZE)

5. **Category Decision Quick Reference** — compact table showing "If your tool does X, it's category Y" for fast lookup during rollout

**Formatting rules:**
- Target 150-250 lines (substantial enough to be useful, not so long that coding agents lose focus)
- Use markdown tables for the mapping, not prose
- Code examples should be copy-pasteable
- Cross-reference aquadif-contract.md for full category definitions (don't duplicate)
- This is a SKILL reference file — write for AI agent consumption (clear, structured, no ambiguity)
