<objective>
Add LLM-driven component selection to the supervisor's ask_user tool, replacing pure regex heuristics with a hybrid approach: rule-based fast path for unambiguous structural context, LLM structured output for ambiguous/novel questions.

Purpose: The rule-based mapper handles 5 explicit patterns but cannot classify ambiguous or novel question patterns. LLM selection enables the supervisor to ask richer interactive questions that map to the correct BioCharm component automatically.

Output: Modified component_mapper.py with hybrid selection, ask_user tool factory with LLM closure, updated graph.py wiring, extended tests.
</objective>
