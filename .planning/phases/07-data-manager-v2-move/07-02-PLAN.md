Add a new step in the `quality-and-tests` job of `.github/workflows/ci-basic.yml`, placed AFTER the "Run linting" step and BEFORE the "Run contract tests" step.

The step checks that no NEW Python files import from the deprecated `lobster.core.data_manager_v2` path. It excludes:
- The shim file itself (`data_manager_v2.py`)
- The `.claude/` directory (not production code)
- Build artifacts (`build/`, `.testing/`)
- Existing test files that use `from ... import` (handled by shim, not worth mass-updating)

```yaml
    - name: Check for deprecated data_manager_v2 imports
      run: |
        # Find new code importing from deprecated path (excludes shim file and build dirs)
        VIOLATIONS=$(grep -rn "from lobster\.core\.data_manager_v2 import" \
          --include="*.py" \
          --exclude="data_manager_v2.py" \
          --exclude-dir=".claude" --exclude-dir="build" --exclude-dir=".testing" \
          --exclude-dir="kevin_notes" --exclude-dir="skills" \
          lobster/scaffold/ packages/ || true)
        if [ -n "$VIOLATIONS" ]; then
          echo "::error::New code must import from lobster.core.runtime.data_manager, not lobster.core.data_manager_v2"
          echo "$VIOLATIONS"
          exit 1
        fi
        echo "No deprecated data_manager_v2 imports found in scaffold/packages"
```

IMPORTANT: The grep scope is intentionally narrow -- only `lobster/scaffold/` and `packages/` (new agent code). The existing 80 source importers in `lobster/` and `tests/` are NOT checked because the shim handles them and mass-updating is deferred to SHIM-01 (v2 requirement). The CI check prevents NEW code from using the old path.
