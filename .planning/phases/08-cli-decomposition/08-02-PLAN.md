**Reduce cli.py to wiring-only (~300-400 LOC):**

cli.py should contain ONLY:
1. Module-level configuration (pandas settings, warnings, multiprocessing -- these MUST stay)
2. `app = typer.Typer(...)` and sub-app definitions
3. `@app.command()` decorated functions with:
   - Full `typer.Option()` / `typer.Argument()` parameter declarations
   - A single lazy import + delegation call in the body
   - Docstring for the command
4. `default_callback()` and `config_callback()` -- Typer callbacks
5. The `app.add_typer(...)` registrations

NO function should have a body longer than ~10 lines (excluding parameter declaration lines). The pattern:
```python
@app.command()
def chat(
    workspace: Optional[Path] = typer.Option(None, "--workspace", "-w", ...),
    # ... parameter declarations ...
):
    """Start an interactive chat session."""
    from lobster.cli_internal.commands.heavy.chat_commands import chat_impl
    chat_impl(workspace=workspace, session_id=session_id, ...)
```

**Update heavy/__init__.py** to include the new modules in `__getattr__` lazy loading if needed (following existing pattern).

**CRITICAL anti-patterns to avoid:**
- Do NOT move Typer decorators or parameter declarations out of cli.py
- Do NOT create circular imports (cli_internal must never import from lobster.cli)
- Do NOT move module-level side effects (pandas config, warning filters) -- keep in cli.py
- The `app` object MUST remain in cli.py (pyproject.toml entry point: `lobster = "lobster.cli:app"`)
